# Medical Recommendations API

Бэкенд для сервиса медицинских рекомендаций пакетов услуг после прохождения опроса.

Подробное описание модели данных и ERD-диаграмма: [ARCHITECTURE.md](ARCHITECTURE.md).

## Что реализовано

- JWT-аутентификация администратора.
- Админские CRUD-эндпоинты для пакетов услуг, вопросов и вариантов ответов.
- Публичная выдача активного опроса.
- Публичная отправка ответов пользователя и расчет рекомендованного пакета.
- Выдача нескольких подходящих пакетов в порядке релевантности.
- Структурированный каталог услуг и ссылки оформления пакетов.
- Redis-кеш публичной анкеты с автоматическим сбросом после изменений.
- Хранение заявок/прохождений опроса.
- SQLAlchemy-модели, Alembic-миграция, Pytest-тесты, Docker Compose.

## Логика рекомендации

У каждого ответа может быть несколько эффектов: каждый эффект начисляет `score` медицинскому индикатору. После отправки анкеты API суммирует баллы по индикаторам, применяет правила пакетов и возвращает подходящие пакеты в порядке релевантности. Вопросы и ответы напрямую с пакетами не связаны.

## Кеширование

Публичный эндпоинт `GET /api/v1/public/questions` кешируется в Redis под ключом `public:questions:v1`. TTL по умолчанию составляет 600 секунд. После изменения вопросов или ответов кеш сбрасывается автоматически. Если Redis временно недоступен, API продолжает читать вопросы из PostgreSQL.

## Запуск локально

```bash
cp .env.example .env
docker compose up --build
```

Документация API будет доступна по адресу:

```text
http://localhost:8000/docs
```

Данные администратора по умолчанию:

```text
admin@example.com
admin12345
```

## Основные эндпоинты

- `POST /api/v1/auth/login` - получить JWT-токен администратора.
- `GET /api/v1/auth/me` - проверить текущего администратора.
- `POST /api/v1/admin/packages` - создать пакет услуг.
- `GET /api/v1/admin/packages` - список пакетов.
- `POST /api/v1/admin/services` - создать услугу.
- `GET /api/v1/admin/services` - список услуг.
- `POST /api/v1/admin/packages/{package_id}/services` - добавить услугу в пакет.
- `POST /api/v1/admin/indicators` - создать медицинский показатель для анализа ответов.
- `GET /api/v1/admin/indicators` - список медицинских показателей.
- `POST /api/v1/admin/packages/{package_id}/rules` - добавить правило рекомендации пакета.
- `GET /api/v1/admin/packages/{package_id}/rules` - список правил пакета.
- `POST /api/v1/admin/questions` - создать вопрос с вариантами ответов.
- `GET /api/v1/admin/questions` - список вопросов.
- `POST /api/v1/admin/answers/{answer_id}/effects` - добавить влияние ответа на индикатор.
- `PATCH /api/v1/admin/answer-effects/{effect_id}` - изменить влияние ответа.
- `GET /api/v1/admin/choices` - список прохождений опроса.
- `GET /api/v1/public/questions` - публичный активный опрос.
- `POST /api/v1/public/choices` - отправить ответы и получить рекомендацию.
